Aggregation Query 
• Important for decision-making questions 
• Calculate statistics on groups 
– Result contains statistical (aggregate) summary 
– Calculated for each group defined by group criteria 
• SQL keywords 
– Aggregate functions: the aggregates that need to be calculated on groups 
– GROUP BY: define groups 
– HAVING: filter groups

What is SQL Aggregation? 
• Data Summarization 
– Aggregation is the process of 
performing a calculation on a set of 
values to return a single, meaningful 
value. 
• Insight Generation 
– It allows you to transform massive 
datasets into high-level insights like 
total revenue, average costs, or 
customer counts. 
• Common Use Cases 
– Finding the total sales for the month, 
identifying the highest-priced product, 
or counting active users. 
3 
Concept: Many inputs, one summary output for each group.

Name Description

The GROUP BY Clause 
The GROUP BY statement organizes 
rows that have the same values into 
summary rows. It is the bridge 
between raw data and aggregate 
calculations. 
 
6 
SELECT department, COUNT(*) 
FROM employees 
GROUP BY department; 
* All non-aggregated columns in the SELECT list must appear in the GROUP BY clause. 

--- PAGE 7 ---
Filtering Groups with HAVING 
• WHERE vs. Aggregate Functions 
– The WHERE clause filters individual 
rows before they are grouped. It 
cannot be used with aggregate 
functions like SUM() or AVG(). 
• The HAVING Clause 
– HAVING was introduced specifically 
to filter groups after the aggregation 
has been performed. It allows you 
to set conditions on the summary 
results. 
 
7 
SELECT department, AVG(salary)  
FROM employees  
GROUP BY department  
HAVING AVG(salary) > 50000; 
 
-- Filters for departments where the average salary exceeds 50k

Query Example 
List the number of students enrolled in each 2017 course offering showing 
the offer number and number of students in the result. 
 SELECT Offering.OfferNo,  
        COUNT(*) AS NumStudents  
  FROM Enrollment, Offering   
  WHERE Offering.OfferNo = Enrollment.OfferNo  
    AND OffYear = 2017 
  GROUP BY Offering.OfferNo;

Query Example 
List the offering number, course number, and average GPA.  Only include 
courses offered in fall 2016 in which the average GPA is greater than 3.0.  
SELECT Enrollment.OfferNo, CourseNo,  
       AVG(StdGPA) AS AvgGPA 
 FROM Offering, Enrollment, Student 
 WHERE Offering.OfferNo = Enrollment.OfferNo 
   AND Student.StdNo = Enrollment.StdNo 
   AND OffYear = 2016 AND OffTerm = 'FALL'  
 GROUP BY Enrollment.OfferNo, CourseNo 
 HAVING AVG(StdGPA) > 3.0;
